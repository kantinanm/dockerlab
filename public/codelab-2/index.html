Git เป็น Distributed Version Control System (DVCS) ซึ่งต้องมีการก็อปปี้เวอร์ชันของซอร์สโค้ดมาเก็บไว้ในเครื่องของเราก่อน (Local Host) ทำให้ผู้ใช้สามารถแก้ไขโปรเจ็กต์ได้ทุกที่แบบ Offline และ Check-in เพื่อจัดเก็บความเปลี่ยนแปลงของซอร์สโค้ดลงในฐานข้อมูลภายในเครื่องของเรา (Local Repository) โดยไม่จำเป็นต้องติดต่อกับ Git Repository บนเซริฟเวอร์และพอแก้ไขเสร็จแล้วจึงค่อยสั่ง Sync เพื่อให้ Version ฝั่ง Local และ Server อัพเดตเหมือนกันในภายหลัง (ด้วยการ Pull/Merge/Push)

การทำงานของ Git

หลักการทำงานของ Git จะแบ่งสถานะของไฟล์ที่มันติดตาม (Tracked) ออกเป็น 3 สถานะ ได้แก่

  • Modified หมายถึง เราได้แก้ไขไฟล์ไปแล้ว แต่ยังไม่เริ่มกระบวนการจัดเก็บลง Repository
  • Staged หมายถึงเราได้ทำเครื่องหมาย File ที่ได้ถูกแก้ไขเพื่อจะบันทึกในเวอร์ชันหน้า
  • Committed หมายถึง ข้อมูลถูกบันทึกอย่างปลอดภัยใน Repository บน Local Host ซึ่งจะแสดงการเปลี่ยนสถานะของไฟล์ในส่วนต่างๆ ดังรูปที่ 1

รูปที่ 1 การทำงานของ Git

จากรูปที่ 1 การทำงานของ Git มีกระบวนการดังนี้

  • เราจะแก้ไขไฟล์ใน Working Directory (ไฟล์ที่แก้ไขจะอยู่ในสถานะ Modified)
  • รวบรวมไฟล์ที่ถูกแก้ไขลงใน Staging Area เพื่อจะ commit ใน Version ถัดไป โดยไฟล์ที่ถูกรวบรวมจะอยู่ในสถานะ Staged
  • Commit เพื่อนำไฟล์จาก Staging Area ไปจัดเก็บอย่างถาวรใน Local Repository (ไฟล์ที่ถูกบันทึกลง Repository อยู่ในสถานะ Committed

การติดตั้ง Git Client และการเรียกใช้งาน

ก่อนจะใช้งาน Git จำเป็นจะต้องมีการติดตั้ง Git Client ก่อน โดยสามารถดาวน์โหลดตัว setup ได้ที่ https://git-scm.com/downloads เลือกดาวน์โหลดตาม OS ที่ใช้งาน ดังรูปที่ 2

รูปที่ 4 ขั้นตอนการ Install Git

git version

ผลลัพธ์ที่ได้จากการเรียกดูเวอร์ชัน ดังรูปที่ 5

รูปที่ 5 ผลลัพธ์ที่ได้จากการเรียกดูเวอร์ชัน

คำสั่ง Git เบื้องต้น

git init

เป็นคำสั่งที่ใช้สร้าง .git ใน Project ของเราซึ่งมันจะสร้าง History เก็บการเปลี่ยนแปลงทั้งหมดที่เราทำใน Project นั้นๆ เมื่อสร้าง Project ใหม่ทุกครั้งสิ่งที่ควรทำเป็นอย่างแรก

git config --global user.name "your-name"
git config --global user.email "your-email@mail.com"

เป็นคำสั่งที่คอนฟิก git ให้จำ username และ email จะใช้คำสั่งนี้ตอนใช้งานครั้งแรก

git add

คำสั่งใช้สำหรับ Track ไฟล์และส่งมันไปยัง Staging Area

git status

คำสั่งใช้สำหรับดูสถานะ Project

git reset

คำสั่งในการย้อนเวอร์ชันไฟล์ให้กลับไปอยู่ในสภาพเดิมก่อนที่จะ add เข้าสู่ Staging Area

git remote add origin https://GitHub.com/username/repository.git

คำสั่งเชื่อมโยง Local Repository บนเครื่องของเรากับ Remote Repository บน GitHub Server 

git commit -m "Description of commit"

เป็นคำสั่งในการจัดเก็บไฟล์อย่างถาวรด้วยการสร้างเป็นเวอร์ชันของซอร์สโค้ด และใช้พารามิเตอร์ -m สำหรับใส่ข้อความช่วยจำ เพื่ออธิบายการ Commit แต่ละเวอร์ชัน

git push

ใช้สำหรับ Sync ข้อมูลกับ GitHub Server 

git pull

ใช้ดึงการเปลี่ยนแปลงล่าสุดจาก remote repository มายัง local repository 

git clone https://GitHub.com/username/repository.git

คำสั่งสำหรับการดึงข้อมูล repository หรือ code ของคนอื่นมาใช้หรือศึกษา

git branch

คือคำสั่งที่ใช้ในการจัดการ branch ต่างๆ 

git checkout branchName

คือคำสั่งในการเปลี่ยนไปยัง branch ที่ระบุ

git merge branchName

คือคำสั่งที่ใช้ในการรวมไฟล์จาก branch ที่ต้องการเข้ามาใน master branch

git log

ใช้ในการเรียกดู History ของ Project ว่ามีการเปลี่ยนแปลงอะไรไปบ้าง

GitHub คือ เว็บไซต์ที่ให้บริการ Git โดยจะให้บริการบนออนไลน์แพลตฟอร์มทำให้เราสามารถเข้าถึงข้อมูลผ่านหน้าเว็บไซต์ได้ทุกที่ทุกเวลา และเรายังสามารถใช้ Git ร่วมกับคนอื่นได้อีกด้วย

การสร้าง Remote Repository

เพื่อจะใช้งาน Distributed Version Control System อย่าง Git Local Host เราจะต้องสร้างการเชื่อมต่อเพื่อติดต่อกับ Remote Repository โดยใช้ URL จาก Clone with HTTP ดังรูปที่ 12 ซึ่ง URL ที่ได้เป็นเหมือนเส้นทางติดต่อระหว่างเครื่องของเรากับ GitHub Server

รูปที่ 12 การก็อปปี้ URL เพื่อใช้ในการเชื่อมโยงกับ Remote Repository

การคอนฟิก Git ให้พร้อมใช้งาน

เปิด File Explorer ขึ้นมาแล้วไปยัง Directory ที่จะใช้สำหรับเก็บ Local Project บนเครื่องของเรา จากนั้นคลิกพื้นที่ว่าง บนช่อง Address Bar ดังรูปที่ 13 จะมีแถบสีน้ำเงินคลุมอยู่ให้เราลบโดยปุ่ม Delete แล้วพิมพ์ cmd ลงไป เสร็จแล้วกด Enter

รูปที่ 13 Address Bar ของ File Explorer

รูปที่ 14 หน้าต่าง Command Prompt

ในการใช้งานครั้งแรก เราจะต้องคอนฟิก Git ให้จำ Username และ Email ด้วยคำสั่ง 

git config --global user.name "Username"
git config --global user.email "email@mail.com"

จากนั้นสร้างโฟลเดอร์ของ Project ชื่อ "gitLab" บน Local Host และใช้คำสั่ง git init เพื่อเริ่มต้นการทำงานจะได้ผลลัพธ์ดังรูปที่ 15

mkdir gitLab //สร้างโฟลเดอร์ของ Project
cd gitLab //เข้า Directory ของโฟลเดอร์ที่เราสร้าง
git init

รูปที่ 15 ผลลัพธ์ที่ได้จากคำสั่ง git init

หลังจากนั้นให้เชื่อมโยง Local Repository บนเครื่องของเรากับ Remote Repository บน GitHub Server โดยนำ URL จาก Clone with HTTP (จากรูปที่ 12) มาใช้ผ่านคำสั่ง 

git remote add origin https://GitHub.com/username/gitLab.git

เป็นอันเสร็จสิ้นสำหรับการคอนฟิก Git เพื่อให้พร้อมใช้งาน

วิธีการ Check-in กับ Local Repository

ให้เราสร้างไฟล์ hello.html ขึ้นมา 1 ไฟล์ โดยใช้โปรแกรม Text Editor ที่มีอยู่ในเครื่อง แล้วก็อปปี้ Code ดังต่อไปนี้ไปวางในไฟล์ hello.html

<HTML>
<HEAD></HEAD>
<BODY>
Hello Git
</BODY>
</HTML>

ไฟล์ที่สร้างใหม่จะยังไม่ถูกติดตามความเปลี่ยนแปลง (track) จากระบบ Git ซึ่งเราสามารถดูสถานะของไฟล์ใน Project ได้โดยใช้คำสั่ง 

git status

สังเกตว่าไฟล์ที่มีสถานะ Modified จะเป็นสีแดง ดังรูปที่ 16

รูปที่ 16 ไฟล์ hello.html ที่ยังไม่ถูก Track

Track ไฟล์ hello.html โดยใช้คำสั่ง git add เพื่อให้ไฟล์อยู่ในสถานะ Staged จะเป็นสีเขียว ดังรูปที่ 17

git add .

รูปที่ 17 ไฟล์ hello.html ที่อยู่ในสถานะ Staged

หลังจากนั้นใช้คำสั่ง git commit เพื่อจัดเก็บไฟล์อย่างถาวรด้วยการสร้างเป็นเวอร์ชันของซอร์สโค้ด และใช้พารามิเตอร์ -m สำหรับใส่ข้อความช่วยจำ เพื่ออธิบายการ Commit แต่ละเวอร์ชัน

git commit - m "Description of commit"

และ Sync History กับ Remote Git Repository โดยใช้คำสั่ง

git push -u origin master

จากนั้นจะมีหน้าต่างยืนยันตันตนกับ GitHub Server ให้คลิกที่ Sign in with your browser จากนั้นคลิก Authorize GitCredentialManager ดังรูปที่ 18 *ในขั้นตอนนี้หากเคย Auth แล้วจะไม่มีหน้าต่างนี้ขึ้นมา

รูปที่ 18 หน้าต่างยืนยันตันตนกับ GitHub Server

เมื่อเข้าไปดูที่ GitHub จะพบว่ามีการ Sync ข้อมูลกับ Remote Repository เรียบร้อยแล้ว ดังรูปที่ 19 ในครั้งต่อไปเราสามารถ Push โดยใช้เพียงคำสั่ง git push สั้นๆ เพราะเราได้ใช้พารามิเตอร์ -u เพื่อให้มันจดจำว่าเราจะ push ไปที่ Remote และ Branch

รูปที่ 19 หน้าจอ Project Detail ที่มีการ Sync ข้อมูลกับ Remote Repository แล้ว
